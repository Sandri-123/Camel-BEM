BEM y CamelCase

BEM significa Block Element Modifier. Es una convención para nombrar clases CSS que ayuda a organizar y mantener el código fácilmente escalable.

¿Qué es BEM?

BEM es una metodología de nomenclatura popular para clases en HTML y CSS. Desarrollado por el equipo de Yandex, El objetivo de BEM es dar mucha más organización, transparencia y claridad en tu estructura HTML y CSS. BEM te dice cómo se relacionan las clases entre sí, lo que es particularmente útil en secciones complejas del documento. Es allí donde se divide en (Bloque, Elementos y Modificadores) entra en escena y se convierte en un aliado para todo Desarrollador Web.

• Ventajas de usar BEM

  1. Mayor claridad y legibilidad del código Los nombres de clases indican explícitamente la función y la estructura del componente.
  2. Reutilización de componente Los bloques se pueden reutilizar en diferentes contextos sin conflictos con otros estilos.
  3. Evita colisiones de nombres Gracias a su estructura jerárquica, se reducen los conflictos entre clases CSS.
  4. Facilita el mantenimiento del código Es más fácil localizar y actualizar estilos relacionados con un componente específico.
  5. Compatible con metodologías modernas (Atomic Design, CSS-in-JS, etc.) BEM se puede combinar con herramientas y metodologías modernas sin problema.
  6. Facilita el testing automatizado Los selectores CSS son predecibles y únicos, lo que los hace ideales para pruebas automatizadas.

¿Qué es camelCase?

El estilo Camelcase es un estilo de mayúsculas y minúsculas donde las palabras se unen sin espacios, y cada palabra nueva empieza con mayúscula, excepto la primera. El nombre "camelCase" proviene de la apariencia abombada de las mayúsculas. Camelcase se usa comúnmente en JavaScript, Java y C# para nombres de variables y funciones.

¿Cómo funciona?

Esto se basa en la idea de concatenar palabras para formar identificadores significativos. Al unir varias palabras en un solo identificador, se facilita la lectura y comprensión del código. Veamos un ejemplo para que lo entiendas con facilidad:

Situaciones donde se usa camelCase

  1. Nombres de variables y funciones en JavaScript
  2. Nombres de elementos o modificadores en BEM
  3. Propiedades de objetos en JavaScript
  4. Parámetros en funciones
  5. Archivos JS exportados como funciones o constantes

Ejemplos de código usando camelCase

  • let userName = "Ana";
  • function obtenerEdadUsuario() {}
  • const nombreCompleto = "Carlos Pérez";
  • usuario.obtenerDatos();
