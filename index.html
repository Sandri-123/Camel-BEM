Aprende las mejores prácticas para nombrar clases y variables en desarrollo web

¿Qué es BEM?

BEM es una metodología de nomenclatura popular para clases en HTML y CSS. Desarrollado por el equipo de Yandex, El objetivo de BEM es dar mucha más organización, transparencia y claridad en tu estructura HTML y CSS. BEM te dice cómo se relacionan las clases entre sí, lo que es particularmente útil en secciones complejas del documento. Es allí donde se divide en (Bloque, Elementos y Modificadores) entra en escena y se convierte en un aliado para todo Desarrollador Web.

Bloques, elementos y modificadores:

  • Bloque: Es una sección independiente que tiene significado propio por sí solo. Contiene todos los nodos HTML de una estructura a la que te estés refiriendo. Se define con nombre descriptivo en minúsculas y sin espacio. Cuando la clase se refiera solo al bloque, será bloque
  • Elemento: Son porciones más pequeñas internas a un bloque. Se usa para ir dividiendo el bloque en segmentos más pequeños. Utiliza el nombre del bloque seguido por el elemento. Cuando se refiera a un elemento dentro de un bloque, sera: bloque_elemento
  • Modificador: sirve para modificar algunas propiedades de un bloque o elemento. Además representa una variante o estado diferente de un bloque o elemento. Se separa del bloque o elemento por dos guiones bajos. Se agrega el bloque o elemento con el que está asociado. sera: bloque__elemento_modificdor

¿Qué es CamelCase?

CamelCase es un estilo de escritura donde cada palabra comienza con mayúscula (excepto la primera, en lowerCamelCase), y no se usan espacios ni guiones.

  • lowerCamelCase: usado en variables y funciones en JavaScript. Ej: nombreUsuario
  • UpperCamelCase (PascalCase): usado en nombres de clases o componentes. Ej: MiComponente
